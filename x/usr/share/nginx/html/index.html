Welcome to NMOX

What is NMOX?

NMOX (New Media On X) is a meta-framework that bridges the gap between traditional UNIX philosophy and modern web development. It builds on a central X object across multiple programming languages, providing a unified approach to web development with Docker-centric deployment.

Key features include:

  • UNIX-inspired architecture and file system structure
  • Multi-language support (C, Perl, Go, Dart, Ruby, Python, PHP, Clojure, and Rust)
  • Docker-based deployment and development environment
  • Built-in support for modern web standards

Nginx

Nginx (pronounced "engine-x") is a high-performance HTTP server, reverse proxy, and load balancer that serves as NMOX's default static file server. Selected for its exceptional efficiency and small footprint, Nginx excels at serving static content and can handle thousands of concurrent connections with minimal resource usage. In NMOX's Docker implementation, Nginx runs on Alpine Linux, resulting in a lightweight container that's perfect for both development and production environments. While NMOX supports other web servers, Nginx's UNIX-like configuration style and robust feature set make it an ideal default choice for serving NMOX applications.

NMOX Server works seamlessly with Nginx. It is intended that you fire up the Docker container and start serving your static files immediately. You can find the Dockerfile for Nginx in x/Docker/nmox-nginx.Dockerfile

Quick Start with Docker

To get started with NMOX's static file serving capability:

  1. Clone the NMOX repository to your local machine
  2. Navigate to the project directory
  3. Build the Docker image with: 
    docker build -t nmox-static .
  4. Run the container with: 
    docker run -d -p 8080:80 --name nmox-static nmox-static

Your static files will be served from x/usr/share/nginx/html within the container. To serve your own static files, you can mount a local directory:

docker run -d -p 8080:80 -v $(pwd)/static:/usr/share/nginx/html --name nmox-static nmox-static

Philosophy

NMOX embodies three core principles:

  • Peace: Represented through Strings - resolution and clarity in code
  • Love: Represented through the X object - finding commonality across different languages and systems
  • Harmony: Represented through Arrays - bringing different components together
